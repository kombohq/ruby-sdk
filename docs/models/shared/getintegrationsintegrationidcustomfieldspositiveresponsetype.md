# GetIntegrationsIntegrationIdCustomFieldsPositiveResponseType

The type of the integration field. There is a type to avoid collisions between DEFAULT fields and CUSTOM fields with the same key.

## Example Usage

```ruby
require "kombo"

value = GetIntegrationsIntegrationIdCustomFieldsPositiveResponseType::DEFAULT
```


## Values

| Name      | Value     |
| --------- | --------- |
| `DEFAULT` | DEFAULT   |
| `CUSTOM`  | CUSTOM    |
